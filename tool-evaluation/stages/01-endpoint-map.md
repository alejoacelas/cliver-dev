# Stage 1 — Endpoint-to-measure map (static)

This file is the hardcoded starting point for the pipeline. It defines which endpoints we're testing and which KYC steps each serves. **This is not generated by an agent** — it's curated based on the previous pipeline's research (103 ideas across 20 measures) and filtered for relevance to the 5 priority KYC steps.

## The 5 KYC steps and their flags

| Step | ID | Measure(s) | Flag |
|---|---|---|---|
| Address → institution | `a` | M05 | No public association between affiliation and shipping address |
| Payment → institution | `b` | M12, M10 | Billing address not associated with the institution; gift card BIN |
| Email → affiliation | `c` | M02, M07 | Does not match institution domain / non-institutional domain |
| Residential address | `d` | M04 | Residential address |
| PO box / freight forwarder | `e` | M03, M06 | P.O. Box; freight forwarder address |

## Endpoints by KYC step

### (a) Address → institution

| Endpoint | What it tells us | Source idea |
|---|---|---|
| **ROR API v2** | Institution's city, country, domains. ~110K orgs. City-level match only. | m05-ror-gleif-canonical |
| **GLEIF API** | Legal entity's registered street address. ~2.9M entities. Biased toward financial/commercial. | m05-ror-gleif-canonical |
| **Companies House** | UK company registered address, SIC codes, dissolved status. ~5.5M UK entities. | m05-ror-gleif-canonical, m09-corp-registry-stack |
| **OpenCorporates** | Corporate registry data across 140+ jurisdictions. Extends Companies House globally. | m09-corp-registry-stack |
| **OSM Overpass** | Campus polygon containment. Can check if address falls within institution boundary. | m05-ror-gleif-canonical |
| **GeoNames** | Reverse geocoding, nearby feature lookup. Campus-center coordinates via ROR's geonames_id. | m05-ror-gleif-canonical |
| **Google Places (New API)** | Nearby Search: find institutions within radius of address. `primaryType` classification. | m04-google-places-business, m05-google-places-campus |
| **Smarty US Street API** | Address normalization for comparison. Also feeds steps (d) and (e). | m05-ror-gleif-canonical |
| **NIH RePORTER** | Institution has federal research funding — strong legitimacy signal. ~2,500 US biomedical institutions. | m18-nih-reporter |
| **NSF Awards / UKRI / CORDIS** | International funding history. ~15-18K institutions across US/UK/EU. | m18-nsf-awards |
| **PubMed affiliation search** | Institution appears in publication affiliations — confirms life-sciences activity. | m09-pubmed-affiliation |
| **LLM+Exa web search** | Standalone: "Does institution X have a presence at address Y?" | llm-exa-search-standalone |

### (b) Payment → institution

| Endpoint | What it tells us | Source idea |
|---|---|---|
| **Stripe test mode** | Card metadata (funding type: credit/debit/prepaid), test AVS codes. Schema validation only. | m10-stripe-funding, m12-psp-avs |
| **Stripe AVS (prod, docs-only)** | Live AVS varies by issuer/country. Documentation review for coverage assessment. | m12-psp-avs |
| **Plaid sandbox** | Identity Match schema. Synthetic data — schema validation only. | m12-psp-avs |
| **Plaid prod (docs-only)** | Production Identity Match: per-field match scores. US bank accounts only. $0.20-$1.00/call. | m12-psp-avs |
| **binlist.net** | Free BIN lookup — card brand, type (debit/credit), bank name. Missing many fintech BINs. | m10-binlist-stack |
| **Billing-shipping-institution consistency** | Triple-address comparison: billing vs. shipping vs. canonical institution address. Local logic + Smarty. | m12-billing-shipping-consistency |
| **Fintech-neobank BIN denylist** | Static list: Mercury, Brex, Relay, Wise BIN ranges. Flags neobank-funded entities. | m12-fintech-denylist |
| **LLM+Exa web search** | Standalone: "Is billing entity X associated with institution Y?" | llm-exa-search-standalone |

### (c) Email → affiliation

| Endpoint | What it tells us | Source idea |
|---|---|---|
| **ROR API v2** | Institution's known domains (e.g., mit.edu). Direct match against customer email domain. | m02-ror-domain-match |
| **RDAP/WHOIS** | Domain registration date, registrant (often privacy-proxied), transfer history. | m02-rdap-age |
| **Disposable/free-mail blocklist** | Static hashset: reject Mailinator/10minutemail; soft-flag Gmail/Yahoo/Outlook. | m02-disposable-blocklist |
| **MX/SPF/DMARC tenant check** | DNS queries: MX resolution, SPF/DMARC records, Microsoft GetUserRealm tenant detection. | m02-mx-tenant |
| **Lookalike/homoglyph domain detector** | Local logic against ROR domain corpus: catches typosquat/impersonation domains. | m18-lookalike-domain |
| **InCommon/eduGAIN federation** | Academic federation IdP list — confirms institution is in the federated identity system. | m07-incommon-edugain |
| **ORCID** | Employment/education records. 2% institution-verified (near-conclusive), 98% self-asserted. | m19-orcid-employments |
| **OpenAlex** | Author affiliation history with ROR IDs, h-index, topic tags. 114M authors. Largest setup effort. | m19-openalex-author |
| **LLM+Exa web search** | Standalone: "Does email domain X belong to institution Y?" | llm-exa-search-standalone |

### (d) Residential address

| Endpoint | What it tells us | Source idea |
|---|---|---|
| **Smarty US Street API** | RDI field: Residential vs. Commercial classification. US addresses only. | m04-usps-rdi |
| **Google Places (New API)** | `primaryType` classification: `university`, `apartment_building`, `coworking_space`, etc. International. | m04-google-places-business |
| **LLM+Exa web search** | Standalone: "Is address X a residential or business/institutional address?" | llm-exa-search-standalone |

### (e) PO box / freight forwarder

| Endpoint | What it tells us | Source idea |
|---|---|---|
| **PO Box regex** | Local pattern matching: PO Box, APO/FPO, PMB. Zero-cost, zero false negatives on standard formats. | m03-pobox-regex-sop |
| **Smarty US Street API** | CMRA flag (Commercial Mail Receiving Agency = mail forwarding). DPV vacancy flag. US only. | m03-smarty-melissa |
| **BIS Country Group D/E + EAR matrix** | Local logic: map destination country to BIS licensing requirements. Export control compliance. | m06-bis-country-groups |
| **ISO 3166 country normalization** | Local logic: normalize country names/codes, check against sanctioned territory list. | m06-iso-country-normalize |
| **Consolidated Screening List** | OFAC SDN, BIS Entity List, DPL, UVL, MEU. Catches denied parties/entities at the address. | m06-bis-entity-list |
| **LLM+Exa web search** | Standalone: "Is address X a PO box, mail forwarding service, or freight forwarder?" | llm-exa-search-standalone |
