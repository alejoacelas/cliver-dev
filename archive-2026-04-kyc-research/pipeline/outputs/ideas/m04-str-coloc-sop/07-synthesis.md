## Section 2: Narrative

### What this check is and how it works

This is a manual reviewer SOP, not an automated API check. It is triggered as a second-line escalation when an upstream M04 check (USPS RDI, county assessor parcel lookup, or Google Places) flags a shipping address as residential or ambiguous. The reviewer manually searches Airbnb.com and VRBO.com for active listings at or near the flagged address, optionally cross-references against Inside Airbnb's quarterly CSV data for covered cities, and queries the Wayback Machine CDX Server for historical domain activity on the customer's claimed institution. If a current STR listing co-locates with the shipping address, the reviewer raises a hard flag (`address_is_str`) and requests lease documentation from the customer. Historical-only STR presence triggers a softer flag with customer Q&A. The SOP takes 10–20 minutes per case and is designed for deep investigation of a small number of flagged addresses, not bulk screening.

### What it catches

The SOP's effective hit surface is narrow by design. It targets the specific bypass method where an attacker uses an active Airbnb or VRBO short-term rental as a drop address. This method is explicitly enumerated in the foreign-institution story (Method 4, residential address via STR at $500–$2,000/month). When an attacker books an Airbnb and ships synthesis orders there, the manual search discovers the active listing. This is the only bypass method the SOP catches with confidence across the mapped attacker stories. For community-bio-lab-network and dormant-domain, the SOP is largely irrelevant because those attackers use their own homes (not STRs) or commercial addresses.

### What it misses

The SOP misses nearly all bypass methods across the mapped stories. Every commercial-address path (virtual offices, coworking, executive suites, maker spaces, incubator benches) is outside its scope — the SOP is not triggered for non-residential addresses. Every residential-own-home path is missed because the attacker's own home is not an STR listing. Non-STR short-term rentals (corporate housing, month-to-month leases) are invisible to Airbnb/VRBO searches. Delisted STRs evade if the attacker removes the listing before screening. Additionally, checking only Airbnb and VRBO misses an estimated 40–50% of the global STR market (Booking.com, regional platforms). The stage 5 hardening assessment classified the narrow hit surface as Moderate — the SOP is correctly scoped for what it does, but what it does is extremely specific.

### What it costs

The dominant cost is reviewer time: $8–17 per check at 10–20 minutes per flagged address. The data sources themselves are free (Airbnb manual search, Inside Airbnb CSV, Wayback CDX) or vendor-gated (AirDNA). Setup cost is minimal — approximately one day for SOP documentation and reviewer training. The high per-check cost means this SOP is economical only when the upstream flag rate is low (perhaps 5–15 cases per day per reviewer). If upstream checks are poorly calibrated and flag too many addresses, reviewer costs escalate quickly.

### Operational realism

The SOP is inherently manual and non-scalable. It requires a trained reviewer to navigate Airbnb's web search, visually compare map pins to the flagged address, cross-reference Inside Airbnb CSVs, and make a judgment call on whether a nearby listing matches. In dense urban areas, the ~150m lat/lng obfuscation in Inside Airbnb data means 5–20+ buildings may fall within the match radius, creating ambiguity that demands reviewer judgment. The Wayback CDX component, while theoretically useful for historical STR detection, is effectively circular for address-based searches: Wayback indexes by URL, so the reviewer needs a listing URL before querying, which they would only have if the Airbnb search already found something. Every disposition is documented with screenshots and decision rationale in the compliance log.

### Open questions

The Inside Airbnb data license for commercial use is unresolved — the site is free for non-commercial use, but whether a synthesis provider's customer screening qualifies as commercial is unclear. The 04C claim check flagged two missing citations: the ~150m obfuscation distance (widely cited in literature but not directly sourced in the implementation) and the claim that Inside Airbnb covers ~100 cities (the actual number appears to be ~165 cities per the coverage research). AirDNA's address-level API pricing is vendor-gated. The Airbnb market share figure (50–60% of global STR market) used to estimate the non-Airbnb coverage gap lacks a direct citation.

## Section 3: Open issues for human review

- **Narrow effective hit surface.** The SOP catches only one specific bypass method (active Airbnb/VRBO STR used as drop address, per foreign-institution Method 4). All other mapped attacker stories either do not use STRs or use the attacker's own residence. This is by design (second-line escalation) but limits the SOP's standalone value.
- **[unknown] Inside Airbnb commercial-use license.** Searched for but not resolved. A synthesis provider using Inside Airbnb data for customer screening may need to confirm licensing.
- **[vendor-gated] AirDNA address-level API pricing.** Public site says "contact sales." AirDNA could partially mitigate Inside Airbnb's geographic limitations but at unknown cost.
- **04C MISSING-CITATION: Inside Airbnb obfuscation distance (~150m).** Widely repeated in literature but not directly cited in the implementation document.
- **04C MISSING-CITATION: Inside Airbnb city count.** The spec says ~100 cities; coverage research found ~165 cities. The implementation document should be corrected.
- **06F flag: Airbnb global STR market share (50–60%)** used to estimate the platform-coverage gap lacks a direct citation.
- **Stage 5 Finding 3: Wayback CDX for address searches is circular.** The CDX server indexes by URL, not by physical address. The Wayback component of the SOP adds value only when the reviewer already has a listing URL from a prior step.
- **Scalability constraint.** At $8–17/check and 10–20 min/case, this SOP is viable only for low-volume escalation. If upstream flag rates are high, reviewer costs become prohibitive.
