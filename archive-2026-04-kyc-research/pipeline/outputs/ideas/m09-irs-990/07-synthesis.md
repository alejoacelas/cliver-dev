## Section 2: Narrative

### What this check is and how it works

This check queries IRS Form 990 filings to verify that a customer claiming US 501(c)(3) nonprofit status has a real, operating organization with life-sciences-relevant programs. The system first searches ProPublica Nonprofit Explorer by organization name or EIN (free, no auth required), then fetches the corresponding Form 990 XML from the IRS public S3 bucket (also free). The 990 contains the organization's total revenue, program-service expenses, mission statement, program descriptions, officer/board lists, and NTEE category code. The system parses these fields and evaluates three flags: `no_990` (no IRS record found), `990_revenue_implausible` (revenue too low for the claimed scope of work, or only 990-N e-postcards filed), and `990_program_not_life_sciences` (NTEE code or mission text does not mention life-sciences activities). Candid (GuideStar) can optionally enrich with financial trends and transparency ratings, but its API pricing is vendor-gated.

### What it catches

The check is narrowly but genuinely useful. For the **shell-nonprofit** story — where an attacker creates a research nonprofit shell — the check can detect nonprofits whose NTEE code or program description does not mention life sciences (`990_program_not_life_sciences`), and nonprofits whose revenue is implausibly low for their claimed research scope (`990_revenue_implausible`). For the **dormant-domain** story (501(c)(3) variant), the check can detect a filing gap: a defunct nonprofit whose most recent 990 is more than 2 years old, signaling that the organization has ceased operations. Within its applicable population, the `990_program_not_life_sciences` flag is one of the more informative signals in the M09 suite because NTEE codes and program descriptions are harder to fake convincingly than incorporation filings — though not impossible.

### What it misses

The check's fundamental limitation is scope: it applies only to US 501(c)(3) organizations that have been operating long enough to file a 990. Seven of the nine attacker stories use for-profit LLCs, making this check structurally inapplicable. Even within the nonprofit space, the shell-nonprofit branch is designed to defeat the check: the attacker can keep gross receipts under $50k (filing only a 990-N e-postcard with no program data), write a convincing life-sciences mission statement on the Form 1023 application (NTEE codes are derived from applicant self-descriptions with no verification), and tolerate the 12–18 month filing lag before any 990 data appears. Foreign nonprofit institutions are entirely outside scope. Fiscally sponsored projects — a legitimate structure for early-stage research — lack their own EIN and produce no match.

### What it costs

The baseline check (IRS S3 + ProPublica) is entirely free. Setup cost is approximately 3 engineering days for the ProPublica integration, S3 index lookup, and 990 XML parser. The XML schema is well-documented by community projects (Nonprofit Open Data Collective tooling, irs990efile R package). Candid enrichment adds vendor-gated pricing (best guess: $5–$50/customer for enterprise contracts) but is optional — the free stack provides the critical fields (revenue, NTEE, program description, officers).

### Operational realism

Because this check applies to roughly 10–20% of synthesis customers at best (US nonprofits with 990 filings), it should be explicitly conditional in the screening workflow — triggered only when the customer claims 501(c)(3) status or is identified as a US nonprofit via other signals. When triggered, the reviewer receives the matched ProPublica record and parsed 990 fields. The most common disposition is `no_990` for customers who are not nonprofits (uninformative, expected). For actual nonprofits, the reviewer reads the program description and NTEE code; a non-life-sciences NTEE adds suspicion but the reviewer must check for generic classification (common among small nonprofits). The `990_revenue_implausible` flag requires judgment: legitimate community biolabs genuinely operate at <$50k revenue, so this flag is noisy for the smallest nonprofits.

### Open questions

The hardening report suggested adding a `990_filing_stale` flag for cases where the most recent 990 is >24 months old, which would automate the dormant-domain filing-gap detection currently dependent on reviewer attention. This was not implemented. The NTEE code and mission statement are attacker-controlled (self-described at 501(c)(3) application), meaning life-sciences NTEE codes should be treated as necessary but not sufficient evidence — an asymmetric treatment that is implicit in the playbook but not explicit. The 04C claim check noted that the Candid API cost estimate ($5–$50/customer) is a best-guess extrapolation, not vendor-confirmed. The 06F form check flagged that the IRS SOI statistics citation for "~80,000–100,000 new determinations/year" is a portal link, not a direct table citation.
